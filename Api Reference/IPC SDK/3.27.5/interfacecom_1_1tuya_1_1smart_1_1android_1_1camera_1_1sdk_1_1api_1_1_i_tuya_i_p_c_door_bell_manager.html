In addition to notifying the client through MQTT protocol messages, doorbell calls may also be pushed, so this interface is provided.

The interface can generate a doorbell call model based on external input parameters to manage subsequent doorbell answering, hanging up, and rejection operations.

Parameters
deviceIdDoorbell device id
messageTypeDoorbell call Message type
msgIdThe unique identifier of the doorbell call message
startTimeDoorbell call start time
cidNodeId of the sub device (for example: gateway sub device), nullable

◆ setDoorbellRingTimeOut()

void com.tuya.smart.android.camera.sdk.api.ITuyaIPCDoorBellManager.setDoorbellRingTimeOut ( int  doorbellRingTimeOut)

Set the doorbell call timeout time.

If the doorbell call is not processed within the specified time, it will call back TuyaSmartDoorBellObserver#doorBellCallDidCanceled to notify and end the doorbell call.

Parameters
doorbellRingTimeOutDoorbell call timeout time, in seconds, the default value is 25s.

◆ setIgnoreWhenCalling()

void com.tuya.smart.android.camera.sdk.api.ITuyaIPCDoorBellManager.setIgnoreWhenCalling ( boolean  ignoreWhenCalling)

When receiving a doorbell call from a device, set whether to automatically ignore subsequent calls from this device.

Parameters
ignoreWhenCallingTrue: If a device is receiving a doorbell call, it will automatically ignore subsequent calls from this device; false: Do not ignore subsequent calls from the same device. The default value is true.
